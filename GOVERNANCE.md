# Governance

Project Sheshnaag is currently maintained by its primary maintainer with review support from trusted contributors as the community grows.

## Decision Making

- Security boundary changes require maintainer approval.
- Public-facing workflow changes require tests and documentation.
- New integrations must preserve the defensive-only product boundary in `docs/SHESHNAAG_SAFETY_POLICY.md`.
- Release candidates require passing CI, dependency audit, and the release checklist in `docs/OSS_PROGRAM_READINESS.md`.

## Maintainer Responsibilities

- Triage issues and PRs.
- Review dependency and security advisories.
- Maintain release notes and changelog entries.
- Keep safety policy, contribution guidance, and maintainer automation workflows current.

## Contributor Path

Contributors should start with issues labeled `good first issue`, documentation improvements, tests, connector hardening, or safe demo corpus improvements. Larger feature work should start with an issue or design note.
