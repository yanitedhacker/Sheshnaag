# Security Policy

Project Sheshnaag is a defensive vulnerability research and validation platform. Please report suspected vulnerabilities privately before opening public issues.

## Supported Versions

Security fixes target the `main` branch and the latest tagged release. Older snapshots are best-effort only unless a maintainer explicitly marks them supported.

## Reporting A Vulnerability

Email the maintainer listed in `MAINTAINERS.md` or open a private GitHub security advisory if the repository has advisories enabled. Include:

- affected version or commit SHA
- reproduction steps
- impact and affected trust boundary
- logs, stack traces, or screenshots when safe
- whether any secret, exploit material, or third-party target data is involved

Do not include live exploit chains, credentials, malware samples, or third-party target details in public issues.

## Response Targets

- Acknowledge: 3 business days
- Initial triage: 7 business days
- Fix or mitigation plan: based on severity and exploitability

## Safe Harbor

Good-faith defensive research against code and infrastructure you own or are authorized to test is welcome. Activity against systems you do not own or lack permission to assess is out of scope.
